🎓 KEY CONCEPTS
════════════════════════════════════════════════════════════════════════════

sessionId (Session ID)
  └─ Unique tracking code generated when user starts purchase
  └─ Format: sess_[timestamp]_[random]
  └─ Propagated through: Stripe metadata → n8n → Database → Portal URL
  └─ Purpose: Enable session tracking & idempotence

Webhook Endpoint
  └─ URL that receives events from external services
  └─ Two webhooks in this solution:
    ├─ /webhook/legado-wizard (user initiates purchase)
    └─ /webhook/stripe-webhook-legado (Stripe sends payment events)

PaymentIntent
  └─ Stripe object representing a charge
  └─ Created when user clicks "Buy"
  └─ Confirmed when payment succeeds
  └─ Contains metadata (sessionId, plan, customer email)

Idempotence
  └─ Processing same event twice produces same result
  └─ Example: Send same Stripe webhook twice → Only ONE invoice created
  └─ Ensures reliability if Stripe retries

Signature Verification
  └─ Proves webhook came from Stripe (not a fake)
  └─ Uses cryptographic signature in X-Stripe-Signature header
  └─ Prevents security vulnerabilities
